## Milestone 1 — Каркас проекта

**Цель:** рабочий скелет репозитория, который собирается в бинарник под три ОС.

- [x] Инициализация `go.mod`, структура файлов (`main.go`, `api.go`, `cache.go`, `excel.go`, `progress.go`)
- [x] Подключить `github.com/xuri/excelize/v2`
- [x] README.md + LICENSE (MIT)
- [x] Приветственное окно-баннер при запуске
- [x] Сборка под Windows/Mac/Linux (`GOOS`/`GOARCH`, `-ldflags "-s -w"`)

**Результат:** `go build` проходит на всех трёх платформах, бинарник печатает баннер и завершается.

---

## Milestone 2 — Работа с hh.ru API

**Цель:** изолированный HTTP-слой, покрывающий все нужные эндпоинты, без бизнес-логики сбора.

- [x] HTTP-клиент: таймаут 10с, User-Agent, Accept-заголовки
- [x] Поиск города через `/suggests/areas` (0 / 1 / несколько результатов)
- [x] Список вакансий по профессии (`/vacancies?text=&area=&date_from=&per_page=100`)
- [x] Детали вакансии (`/vacancies/<id>`) и извлечение `key_skills[].name`
- [x] Retry (3 попытки, пауза 2с) + пауза 400мс между запросами

**Результат:** функции `api.go` можно вызвать напрямую и получить корректные данные по любому городу/профессии.

---

## Milestone 3 — CLI-диалог с пользователем

**Цель:** интерактивный сценарий запуска от приветствия до старта сбора.

- [x] Ввод города + обработка неоднозначных совпадений (выбор номера)
- [x] Фиксированный список 20 профессий (константы)
- [x] Интеграция с кэшем: предложение продолжить/начать заново

**Результат:** пользователь проходит весь диалог до момента запуска сбора данных.

---

## Milestone 4 — Кэширование

**Цель:** устойчивость к прерываниям — прогресс не теряется при сбое/закрытии.

- [x] Модель кэша `cache_<город>.json`
- [x] Запись кэша после каждой завершённой профессии
- [x] Чтение/валидация кэша при старте, восстановление прогресса

**Результат:** прерванный на середине сбор корректно продолжается с первой незавершённой профессии.

---

## Milestone 5 — Прогресс в консоли

**Цель:** live-отображение хода сбора без спама в терминал.

- [x] Прогресс-бар `\r` с процентом и счётчиком профессий
- [x] Блок текущего статуса (профессия/вакансия/навыки/время/ETA/ошибки)
- [x] Список завершённых профессий с таймингами

**Результат:** вывод обновляется на месте (без прокрутки), соответствует макету из ТЗ.

---

## Milestone 6 — Генерация Excel

**Цель:** финальный `.xlsx`-отчёт с оформлением, готовый к отправке заказчику.

- [ ] Сборка агрегированной таблицы «навыки × профессии» (сортировка по сумме)
- [ ] Стилизация: шрифты, заливка, границы, ширина колонок
- [ ] Merged заголовок города, freeze panes, альбомная ориентация
- [ ] Запись `<город>_навыки.xlsx` (перезапись существующего)

**Результат:** открытый в Excel файл визуально совпадает со спецификацией стиля из ТЗ.

---

## Milestone 7 — Завершение и edge cases

**Цель:** довести приложение до продакшен-качества на граничных случаях.

- [ ] Итоговый экран статистики
- [ ] Edge cases: пустой `key_skills`, 0 вакансий по профессии, кириллица в имени файла
- [ ] Логирование ошибок/пропусков после retry

**Результат:** полный прогон от старта до `.xlsx` проходит без паники на реальных данных hh.ru.
