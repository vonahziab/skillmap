# ADR-0003: Без токена — но /vacancies через HTML сайта, а не JSON API

## Статус
Принято (пересмотрено — исходное решение использовать JSON-эндпоинты
`api.hh.ru/vacancies` и `api.hh.ru/vacancies/<id>` не сработало на практике)

## Контекст
hh.ru официально документирует `/vacancies`, `/vacancies/<id>`,
`/suggests/areas` как доступные анонимно, без OAuth-токена. Однако при
реализации Milestone 2 выяснилось, что `api.hh.ru/vacancies` и
`api.hh.ru/vacancies/<id>` стабильно возвращают `403 Forbidden` от
`DDoS-Guard` — независимо от заголовков (проверялись разные User-Agent),
IP/сети (проверено с трёх независимых точек: песочница, реальная машина
автора, отдельное устройство в другой сети) и наличия куки реальной сессии
hh.ru. При этом `api.hh.ru/suggests/areas` с тех же точек отвечает `200 OK`
без проблем — блокировка специфична именно для эндпоинтов вакансий,
наиболее популярных у скрейперов.

## Решение
- `/suggests/areas` — используется JSON API как и раньше (не заблокирован).
- Список вакансий и детали вакансии — теперь получаются с обычных HTML-страниц
  сайта (`hh.ru/search/vacancy`, `hh.ru/vacancy/<id>`) с браузерным
  `User-Agent`, а не с `api.hh.ru`. Из HTML вытаскивается не DOM-разметка,
  а встроенный JSON-стейт страницы (`<template id="HH-Lux-InitialState">`) —
  он содержит структуру, эквивалентную официальному API
  (`vacancySearchResult.vacancies[]`, `vacancyView.keySkills`), и не теряет
  вакансии, в отличие от парсинга вёрстки. Этот приём подтверждён на практике
  актуальным сторонним проектом `github.com/LuaSavage/hh_ru_parser`, который
  решает ту же проблему тем же способом.
- Авторизация (OAuth application token) не вводится — с ней не проверяли,
  так как HTML-путь уже даёт `200 OK` без регистрации приложения, сохраняя
  исходную цель ADR («пользователь просто запускает `.exe`»).

## Обоснование
- Сохраняет главный плюс исходного решения — не требует регистрации
  приложения и хранения секретов.
- Не требует обхода анти-бот защиты (капчи, JS-challenge, ротации прокси) —
  обычная браузерная страница отдаётся напрямую, без дополнительных техник.
- Данные из `HH-Lux-InitialState` по составу полей совпадают с тем, что
  вернул бы официальный JSON API — переход не теряет точности.

## Последствия
- Формат ответа определяется вёрсткой сайта, а не версионированным API —
  hh.ru может переименовать `HH-Lux-InitialState` или изменить структуру
  JSON без предупреждения; это более хрупкая зависимость, чем публичный API.
- HTML-страницы значительно тяжелее JSON (сотни КБ на страницу вместо
  нескольких КБ) — незначительно влияет на скорость, не на корректность.
- hh.ru делает редирект с `hh.ru` на региональный поддомен (например
  `ufa.hh.ru`) — клиент обязан сам проходить редирект, сохраняя заголовки на
  каждом хопе (см. `doHTMLFetch` в `api.go`).
- Годовая пауза 400мс между запросами и retry с паузой 2с (раздел 5 ТЗ)
  остаются в силе — блокировка была не про частоту запросов, но лишняя
  осторожность не помешает при таком объёме трафика (до 20 профессий ×
  100 вакансий).
