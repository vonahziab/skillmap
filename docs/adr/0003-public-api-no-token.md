# ADR-0003: Публичное API hh.ru без токена

## Статус
Принято

## Контекст
hh.ru предоставляет открытые эндпоинты (`/vacancies`, `/vacancies/<id>`,
`/suggests/areas`), доступные без OAuth-токена и регистрации приложения.

## Решение
Использовать только публичные, неавторизованные эндпоинты API hh.ru.

## Обоснование
- Убирает шаг регистрации приложения и хранения секретов — конечный
  пользователь просто запускает `.exe`.
- Достаточно для задачи: нужны только текст вакансии, `key_skills`,
  список городов — всё это отдаётся без токена.

## Последствия
- Более строгие анонимные rate-limits → обязательна пауза 400мс между
  запросами и retry с бэкоффом (см. раздел 5 ТЗ).
- Нет доступа к приватным полям вакансий (зарплата работодателя, контакты и
  т.п.) — не требуется по ТЗ.
- Публичное API может менять поведение без предупреждения; версионирования
  нет — риск принимается как часть выбора «без токена».
