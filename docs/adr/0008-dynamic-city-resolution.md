# ADR-0008: Динамическое разрешение города через API вместо хардкода

## Статус
Принято

## Контекст
ID города в hh.ru API — внутренний числовой идентификатор (`area`),
неочевидный пользователю. Список городов велик и не фиксирован в задаче —
пользователь должен иметь возможность ввести любой город.

## Решение
Город вводится пользователем как текст и резолвится в ID через
`GET /suggests/areas?text=<город>`. Обрабатываются три исхода: 0 совпадений
(повторный ввод), 1 совпадение (используется сразу), несколько совпадений
(пользователь выбирает номер из списка).

## Обоснование
- Не требует поддержания собственного справочника городов/ID в коде —
  hh.ru — источник истины, обновляется на их стороне.
- UX проще: пользователь вводит название на естественном языке, а не ищет
  ID вручную.

## Последствия
- Приложение зависит от доступности `/suggests/areas` при каждом запуске —
  без сети старт невозможен (приемлемо, т.к. весь инструмент сетевой).
- Неоднозначные названия (например, «Казань» vs «Казань-на-Волге») требуют
  дополнительного шага диалога — заложено в сценарий (раздел 6, шаг 2 ТЗ).
