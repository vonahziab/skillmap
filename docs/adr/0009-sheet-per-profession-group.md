# ADR-0009: Один лист на группу профессий со своим топ-навыков

## Статус
Принято (заменяет [ADR-0007](0007-single-sheet-output.md))

## Контекст
По [ADR-0007](0007-single-sheet-output.md) все 20 профессий выводились в одну
таблицу, а строки-навыки отбирались как глобальный топ по сумме встречаемости
по всем профессиям сразу. Профессии охватывают три разных мира (разработка /
аналитика / финансы) с сильно разным числом вакансий. Группа с бо́льшим объёмом
вакансий давала бо́льшие абсолютные счётчики, поэтому её навыки вытесняли весь
топ. В результате для профессий из «малых» групп (например, программистов рядом
с бизнес-аналитиками) в колонках стояли почти одни нули — их реальные топ-навыки
не попадали в общий отбор строк.

## Решение
Один лист Excel на группу профессий, а не на весь список. Профессии
сгруппированы в `professionGroups` (`professions.go`): «Разработка»,
«Аналитика и управление», «Финансы и экономика». Для каждой группы строится
свой топ навыков (`aggregateSkills` по профессиям только этой группы),
столбцы — профессии группы, строка 1 — «<Город> — <Группа>». Имя листа =
название группы. Файл по-прежнему один: `<город>_навыки.xlsx`.

## Обоснование
- Внутри группы профессии однородны по объёму вакансий и набору навыков,
  поэтому топ группы репрезентативен для каждой её колонки — нет «мёртвых»
  столбцов из нулей.
- Сравнение навыков между родственными профессиями (то, ради чего строится
  матрица) сохраняется внутри листа; сравнение между разными мирами и так было
  малоосмысленным.
- Группировка уже подразумевалась структурой списка профессий (раздел 4 ТЗ);
  решение делает её явной.

## Последствия
- В книге теперь несколько листов; freeze panes и альбомная ориентация
  применяются к каждому.
- Состав и границы групп захардкожены рядом со списком профессий (см.
  [ADR-0005](0005-fixed-profession-list.md)); изменение требует пересборки.
- Единой сводной таблицы «все профессии сразу» больше нет; при необходимости
  такого вида это отдельная фича.
