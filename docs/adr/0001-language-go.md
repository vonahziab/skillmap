# ADR-0001: Язык реализации — Go

## Статус
Принято

## Контекст
Нужен консольный инструмент, который конечный пользователь (не разработчик)
запускает на Windows 10/11 x64 как один `.exe`, без установки рантайма
или зависимостей.

## Решение
Используем Go 1.21+.

## Обоснование
- Компилируется в один статический бинарник без внешнего рантайма.
- Кросс-компиляция под Windows/Mac/Linux из любой ОС (`GOOS`/`GOARCH`).
- Стандартная библиотека закрывает HTTP-клиент и консольный ввод без
  дополнительных зависимостей.

## Последствия
- Экосистема Excel-библиотек для Go меньше, чем для Python/JS — выбор пал на
  `excelize` (см. ADR-0002).
- Команда должна писать на Go; альтернативы (Python + PyInstaller, Node + pkg)
  дают более тяжёлые и медленные бинарники.
