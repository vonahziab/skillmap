# ADR-0007: Один лист «навыки × профессии» вместо листа на профессию

## Статус
Заменено на [ADR-0009](0009-sheet-per-profession-group.md)

## Контекст
Результат сбора нужно представить так, чтобы можно было сравнивать частоту
навыков между профессиями за один взгляд.

## Решение
Один лист Excel на город: строки — навыки (отсортированы по убыванию суммы
по всем профессиям), столбцы — 20 профессий, ячейки — счётчик встречаемости.
Название листа = название города. Имя файла — `<город>_навыки.xlsx`.

## Обоснование
- Сводная матрица «навыки × профессии» отвечает на главный вопрос ТЗ
  напрямую, без необходимости переключаться между листами и вручную сверять
  строки.
- Сортировка по убыванию суммы выносит самые востребованные навыки в город
  наверх — сразу видна общая картина рынка.

## Последствия
- При росте числа уникальных навыков (сотни строк) лист может стать
  длинным — компенсируется freeze panes (первые 2 строки + первый столбец)
  для удобной прокрутки.
- Если в будущем понадобится вид «один лист на профессию» — это отдельная
  фича, не предусмотренная текущим ТЗ.
