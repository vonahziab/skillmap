# ADR-0004: JSON-кэш на город, запись после каждой профессии

## Статус
Принято

## Контекст
Полный сбор по 20 профессиям × 100 вакансий с паузами 400мс и retry может
занимать 15–20+ минут. Разрыв соединения, закрытие консоли или сбой не
должны обнулять уже собранные данные.

## Решение
После завершения каждой профессии сохранять промежуточный результат в
`cache_<город>.json` (список завершённых профессий + агрегированные
счётчики навыков). При следующем запуске для того же города — предложить
продолжить с первой незавершённой профессии или начать заново.

## Обоснование
- Простой формат (JSON), не требует БД или внешних сервисов — соответствует
  цели «единый .exe без зависимостей».
- Гранулярность «после каждой профессии» — баланс между надёжностью и
  накладными расходами на запись файла (20 записей на весь прогон).
- Кэш привязан к городу (`cache_<город>.json`) — можно параллельно вести
  сбор для нескольких городов, каждый со своим прогрессом.

## Последствия
- Формат кэша — часть публичного контракта; при изменении структуры
  (`data`, `completed`) нужно продумать миграцию или инвалидацию старых
  файлов.
- Не решает конкурентный доступ (два процесса SkillMap на один город
  одновременно) — вне рамок задачи, инструмент однопользовательский.
