# Runbook

Практическое руководство по запуску, восстановлению и диагностике
проблем SkillMap. Для устройства приложения см. [architecture.md](architecture.md).

## Обычный запуск

```
./skillmap.exe
```

1. Ввести название города.
2. Если найдено несколько городов — выбрать номер.
3. Если найден кэш от предыдущего запуска — выбрать «Продолжить» или
   «Начать заново».
4. Дождаться завершения сбора (полный прогон по 20 профессиям занимает
   ориентировочно 15–20+ минут — зависит от количества вакансий и сети).
5. Забрать `<город>_навыки.xlsx` из рабочей директории.

## Восстановление после прерывания

Если процесс был закрыт, завис или упал на середине сбора:

1. Запустить `skillmap.exe` заново для того же города.
2. Приложение найдёт `cache_<город>.json` и покажет, сколько профессий уже
   собрано и когда обновлялся кэш.
3. Выбрать «Продолжить» — сбор начнётся с первой незавершённой профессии,
   уже собранные данные не теряются.

Если кэш повреждён (невалидный JSON) — приложение не сможет его прочитать.
В этом случае: удалить `cache_<город>.json` вручную и начать сбор заново.

## Диагностика типовых проблем

| Симптом | Вероятная причина | Что делать |
|---|---|---|
| «Город не найден» на существующем городе | Опечатка или редкое написание | Ввести часть названия короче, проверить `/suggests/areas` вручную в браузере |
| Сбор аномально долгий, растёт «Ошибок/повторов» | Сетевые проблемы или троттлинг со стороны hh.ru API | Дождаться завершения — пропущенные вакансии логируются, не блокируют остальные; при системных сбоях — перезапустить позже, кэш сохранит прогресс |
| Итоговая `.xlsx` открывается, но профессия показывает одни нули | По профессии не найдено вакансий за 30 дней в этом городе, либо все вакансии без `key_skills` | Ожидаемое поведение — см. edge cases в ТЗ (раздел 10) |
| Приложение зависает на старте | Нет сети / API hh.ru недоступен | Проверить доступность `https://api.hh.ru` вручную, повторить позже |
| `.xlsx` не открывается / повреждён | Процесс был прерван во время записи файла (шаг 5) | Перезапустить сбор для города — кэш позволит не пересобирать данные заново, файл перезапишется |

## Файлы, которые оставляет приложение

| Файл | Назначение | Можно ли удалить после успешного завершения |
|---|---|---|
| `<город>_навыки.xlsx` | Итоговый отчёт | Нет — это результат работы |
| `cache_<город>.json` | Кэш прогресса сбора | Да, если сбор завершён на 20/20 и `.xlsx` уже создан |

## Логи

Ошибки retry и пропущенные вакансии/профессии выводятся в консоль по ходу
сбора и учитываются в итоговом счётчике «Ошибок/повторов» (см. финальный
экран). Отдельный файл логов не предусмотрен текущим ТЗ — при необходимости
разбора конкретного сбоя ориентируйтесь на вывод консоли за время запуска.
