## 1. Общее описание

Консольное приложение на Go. Пользователь вводит любой город — приложение собирает навыки из вакансий hh.ru по 20 профессиям через официальное публичное API (без токена) и сохраняет результат в `.xlsx` файл: одна таблица «навыки × профессии».

- Вакансии только за последние 30 дней
- Максимум 100 вакансий на профессию
- Город не захардкожен — вводится пользователем, ID ищется динамически через API
- Собирается в один `.exe` без зависимостей

## 5. Параметры сбора вакансий

| Параметр | Значение |
|---|---|
| Период | Последние 30 дней (`date_from` = сегодня − 30 дней) |
| Макс. вакансий на профессию | 100 (`per_page=100`, 1 страница) |
| Пауза между запросами | 400 мс |
| Таймаут запроса | 10 секунд |
| Retry при ошибке | 3 попытки с паузой 2 сек |
| Токен | Не требуется (публичное API) |

## 6. Сценарий запуска

### Шаг 1 — Приветствие

При запуске выводится приветственное окно с маркировкой (см. раздел 2).

### Шаг 2 — Ввод города

```
Введите название города: _
```

- Запрос к `https://api.hh.ru/suggests/areas?text=<город>`
- Если найден один → `Город найден: Алматы (ID: 160)`
- Если не найден → просит ввести снова
- Если найдено несколько → показывает список, просит выбрать номер:

```
Найдено несколько городов:
  1. Казань (ID: 88)
  2. Казань-на-Волге (ID: 1234)
Выберите номер: _
```

### Шаг 3 — Проверка кэша

```
Найден кэш от 2026-07-01 14:32 (собрано 12/20 профессий)
Последняя завершённая: BI-аналитик

  1. Продолжить с [Product Manager]
  2. Начать заново

Выберите: _
```

Если кэша нет — сразу переходит к сбору.

### Шаг 4 — Сбор данных

#### Прогресс (обновляется на месте через `\r`):

```
╔══════════════════════════════════════════════╗
║  Город: Москва        Вакансий макс: 100     ║
║  Период: последние 30 дней                   ║
╚══════════════════════════════════════════════╝

Прогресс: [████████████░░░░░░░░] 8/20 (40%)

Текущая профессия : Data Scientist (9/20)
Текущая вакансия  : 67/100
Собрано навыков   : 312 уникальных
Прошло времени    : 10мин 14сек
Осталось (оценка) : ~8мин
Ошибок/повторов   : 3

Завершённые профессии:
  [✓] Backend-разработчик     94 навыка   1м 21с
  [✓] Frontend-разработчик    81 навык    1м 18с
  [ ] Data Scientist          ← сейчас
```

#### Логика сбора:

1. `GET https://api.hh.ru/vacancies?text=<профессия>&area=<id>&per_page=100&page=0&date_from=<дата-30дней>`
2. Для каждой вакансии: `GET https://api.hh.ru/vacancies/<id>`
3. Из деталей берём `key_skills[].name`
4. Считаем счётчик: `map[skill]count`
5. Пауза 400 мс между запросами
6. При ошибке — retry 3 раза с паузой 2 сек, затем пропустить и залогировать
7. Если `key_skills` пустой — пропустить молча

#### Кэш после каждой профессии:

Файл: `cache_<город>.json`

```json
{
  "city": "Москва",
  "city_id": 1,
  "created_at": "2026-07-01T14:32:00Z",
  "updated_at": "2026-07-01T14:44:00Z",
  "completed": ["Backend-разработчик"],
  "data": {
    "Backend-разработчик": { "Python": 45, "Docker": 38 }
  }
}
```

### Шаг 5 — Генерация .xlsx

**Структура файла:**
- Один лист, название = название города
- Имя файла: `<город>_навыки.xlsx`
- Строка 1 (merged): название города
- Строка 2: `Навыки` | профессия 1 | ... | профессия 20
- Строки 3+: навык | число | ... (`0` если навык не встречался)
- Навыки отсортированы по убыванию суммы по всем профессиям

**Стиль:**

| Элемент | Значение |
|---|---|
| Шрифт | Arial, размер 10 |
| Строка города | Фон `#4472C4`, белый текст, жирный, по центру, merged |
| Заголовки профессий | Фон `#4472C4`, белый текст, жирный, по центру |
| Чётные строки | Фон `#DCE6F1` |
| Нечётные строки | Белый фон |
| Колонка «Навыки» | Жирный текст, ширина 25 символов |
| Колонки профессий | Ширина 15 символов |
| Границы | Тонкая рамка у всех ячеек |
| Freeze panes | Первые 2 строки + первый столбец |
| Ориентация | Альбомная |

## 10. Edge cases

- Вакансия без `key_skills` — пропустить молча
- API вернул пустой список — остановить сбор для этой профессии
- Файл `.xlsx` уже существует — перезаписать
- Кириллица в названии файла — использовать как есть
- После 3 retry — записать в лог, продолжить следующую вакансию
- Если по профессии найдено 0 вакансий за период — отметить в итогах
